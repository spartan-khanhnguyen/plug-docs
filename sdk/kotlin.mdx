## Overview

The plugnpay-kt SDK provides a secure, server-side interface for developers to integrate their Kotlin/JVM applications with the PlugnPay service. The library is built with Kotlin Coroutines for modern, non-blocking asynchronous operations and leverages Kotlin's type safety and nullability features to provide a robust developer experience. It is suitable for use in any JVM backend framework like Ktor, Spring Boot, or Micronaut.

**Requirements:** JVM 11+ / Kotlin 1.9+

## Configuration & Initialization

The SDK client is configured via a primary constructor that accepts a configuration object. A helper function is provided for easy initialization from environment variables.

### API Key Authentication

Authentication is handled via an API key that can be configured in two ways:

**Environment Variable (Recommended):**
```bash
export PLUGNPAY_API_KEY="pnp_sk_xxxxxx"
```

**Direct Configuration:**
```kotlin
import com.plugnpay.sdk.PlugnPayClient
import com.plugnpay.sdk.Config

// Preferred: Initialize by reading PLUGNPAY_API_KEY from the environment
val client = PlugnPayClient.fromEnv()

// Alternatively, initialize with explicit config
val config = Config(
    apiKey = "pnp_sk_xxxxxx", // Required
)
val clientWithOptions = PlugnPayClient(config)
```

### Configuration Options

```kotlin
data class Config(
    val apiKey: String, // API key for authentication
)
```

## Core Methods

All methods that interact with the PlugnPay REST API are suspend functions, designed to be called from within a coroutine. They will throw a `PlugnPayException` (or a subclass) on failure. Models are defined using immutable data classes.

### getCustomer

Retrieves or creates a customer record (upsert logic).

```kotlin

    val customer = client.getCustomer(email = "user@example.com")

}
```

## Real-time Event Handling

The SDK provides a Flow-based interface for handling real-time events via Server-Sent Events (SSE). This provides a cold, asynchronous stream of events that integrates seamlessly with coroutines and structured concurrency.

### Flow-Based Interface

The client provides a method that returns a Flow of events.

```kotlin
val eventFlow = client.listenForEvents()
```

**Method:** `fun listenForEvents(): Flow<Event>`

**Behavior:** 
- The Flow is cold; the SSE connection is only established when a terminal operator (like `collect`) is called
- The connection is automatically torn down when the collecting coroutine's scope is cancelled
- Errors in the stream will be thrown as exceptions within the flow, which can be handled with the `.catch` operator

### Event Structure

The SDK uses a sealed class hierarchy for event data to provide maximum type safety when processing events:

```kotlin
import kotlinx.serialization.SerialName
import kotlinx.serialization.Serializable

// Generic event wrapper
@Serializable
data class Event(
    @SerialName("event_id") val eventId: String,
    @SerialName("event_type") val eventType: String,
    @SerialName("created_at") val createdAt: String,
    val data: EventData
)

// Sealed class for type-safe event data
@Serializable
sealed class EventData {
    @Serializable
    @SerialName("subscription_event")
    data class SubscriptionEvent(
        val customer: Customer, 
        val subscription: Subscription
    ) : EventData()

    @Serializable
    @SerialName("invoice_event")
    data class InvoiceEvent(
        val customer: Customer, 
        val invoice: Invoice
    ) : EventData()
}
```

### Event Data Models

```kotlin
@Serializable
data class Subscription(
    val id: String,

    @SerialName("current_period_start") val currentPeriodStart: String,
    @SerialName("current_period_end") val currentPeriodEnd: String,
```


### Advanced Event Handling with Structured Concurrency

```kotlin
import kotlinx.coroutines.*
import kotlinx.coroutines.flow.*

class PlugnPayEventHandler(private val client: PlugnPayClient) {
    private val scope = CoroutineScope(Dispatchers.IO + SupervisorJob())
    
    fun start() {
        scope.launch {
            client.listenForEvents()
                .flowOn(Dispatchers.IO) // Handle network I/O on IO dispatcher
                .catch { exception ->
                    println("Event stream error: ${exception.message}")
                    // Implement retry logic with exponential backoff
                    delay(5000)
                    start() // Restart listening
                }
                .collect { event ->
                    // Process each event in a separate coroutine for concurrency
                    launch { 
                        processEvent(event) 
                    }
                }
        }
    }
    
    fun stop() {
        scope.cancel()
    }
    
    private suspend fun processEvent(event: Event) {
        try {
            println("Processing event ${event.eventId}: ${event.eventType}")
            
            when (val data = event.data) {
                is EventData.SubscriptionEvent -> handleSubscriptionEvent(event.eventType, data)
                is EventData.InvoiceEvent -> handleInvoiceEvent(event.eventType, data)
            }
        } catch (e: Exception) {
            println("Error processing event ${event.eventId}: ${e.message}")
            // Log error, send to monitoring system, etc.
        }
    }
    
    private suspend fun handleSubscriptionEvent(eventType: String, data: EventData.SubscriptionEvent) {
        when (eventType) {
            "subscription.created" -> {
                println("New ${data.subscription.productId} subscription for ${data.customer.email}")
                provisionAccount(data.customer.id)
                sendWelcomeEmail(data.customer.email)
            }
            "subscription.updated" -> {
                println("Subscription updated for ${data.customer.email}: ${data.subscription.status}")
                updateUserPermissions(data.customer.id, data.subscription.status)
            }
            "subscription.deleted" -> {
                println("Subscription cancelled for ${data.customer.email}")
                revokeAccess(data.customer.id)
            }
        }
    }
    
    private suspend fun handleInvoiceEvent(eventType: String, data: EventData.InvoiceEvent) {
        when (eventType) {
            "invoice.payment.succeeded" -> {
                val amount = data.invoice.amountPaid / 100.0
                println("Payment of $${amount} succeeded for ${data.customer.email}")
                sendPaymentReceipt(data.customer.email, data.invoice.url)
            }
            "invoice.payment.failed" -> {
                println("Payment failed for ${data.customer.email}")
                sendPaymentFailedNotification(data.customer.id, data.invoice.url)
                pauseServiceAccess(data.customer.id)
            }
        }
    }
    
    // Helper methods - implement according to your needs
    private suspend fun provisionAccount(customerId: String) {
        println("Provisioning account for customer $customerId")
        // Implement account provisioning logic
    }
    
    private suspend fun sendWelcomeEmail(email: String) {
        println("Sending welcome email to $email")
        // Implement welcome email sending
    }
    
    private suspend fun updateUserPermissions(customerId: String, status: String) {
        println("Updating permissions for customer $customerId to $status")
        // Implement permission updates
    }
    
    private suspend fun revokeAccess(customerId: String) {
        println("Revoking access for customer $customerId")
        // Implement access revocation
    }
    
    private suspend fun sendPaymentReceipt(email: String, invoiceUrl: String?) {
        println("Sending payment receipt to $email")
        // Implement receipt sending
    }
    
    private suspend fun sendPaymentFailedNotification(customerId: String, invoiceUrl: String?) {
        println("Sending payment failed notification to customer $customerId")
        // Implement payment failure notification
    }
    
    private suspend fun pauseServiceAccess(customerId: String) {
        println("Pausing service access for customer $customerId")
        // Implement service access pausing
    }
}

// Usage
```

## Complete Integration Example

```kotlin
import com.plugnpay.sdk.*
import kotlinx.coroutines.*

class PlugnPayService {
    private val client = PlugnPayClient.fromEnv()
    
    suspend fun createCustomerAndCheckout(email: String, productId: String): CheckoutResult {
        return try {
            // Get or create customer
            val customer = client.getCustomer(email)
            println("Customer ID: ${customer.id}")
            
            // Create checkout session
            val session = client.createCheckoutSession(
                CreateCheckoutSessionRequest(
                    customerId = customer.id,
                    productId = productId,
                    successUrl = "https://yourapp.com/success",
                    cancelUrl = "https://yourapp.com/cancel"
                )
            )
            
            CheckoutResult.Success(
                customerId = customer.id,
                checkoutUrl = session.url
            )
        } catch (e: PlugnPayException) {
            CheckoutResult.Error("Checkout creation failed: ${e.message}")
        }
    }
    
    suspend fun checkUserAccess(customerId: String, featureId: String): AccessResult {
        return try {
            val access = client.checkAccess(customerId, featureId)
            
            if (access.hasAccess) {
                // Track usage for metered features
                client.trackUsage(
                    TrackUsageRequest(
                        customerId = customerId,
                        featureId = featureId,
                        value = 1L
                    )
                )
                
                // Get updated usage stats
                val usage = client.getUsage(customerId, featureId)
                
                AccessResult.Granted(
                    usage = usage.totalUsage,
                    remaining = usage.remaining
                )
            } else {
                AccessResult.Denied
            }
        } catch (e: PlugnPayException) {
            AccessResult.Error("Access check failed: ${e.message}")
        }
    }
}

// Result types for better error handling
sealed class CheckoutResult {
    data class Success(val customerId: String, val checkoutUrl: String) : CheckoutResult()
    data class Error(val message: String) : CheckoutResult()
}

sealed class AccessResult {
    data class Granted(val usage: Long, val remaining: Long?) : AccessResult()
    object Denied : AccessResult()
    data class Error(val message: String) : AccessResult()
}

// Usage example
fun main() = runBlocking {
    val plugnPayService = PlugnPayService()
    
    // Create checkout for a customer
    val checkoutResult = plugnPayService.createCustomerAndCheckout(
        email = "customer@example.com",
        productId = "premium-plan"
    )
    
    when (checkoutResult) {
        is CheckoutResult.Success -> {
            println("Redirect to: ${checkoutResult.checkoutUrl}")
            
            // Later, check if user has access to features
            val accessResult = plugnPayService.checkUserAccess(
                customerId = checkoutResult.customerId,
                featureId = "premium-features"
            )
            
            when (accessResult) {
                is AccessResult.Granted -> {
                    println("Feature access granted. Usage: ${accessResult.usage}")
                    accessResult.remaining?.let { remaining ->
                        println("Remaining: $remaining")
                    }
                }
                is AccessResult.Denied -> {
                    println("Upgrade required for this feature")
                }
                is AccessResult.Error -> {
                    println("Error: ${accessResult.message}")
                }
            }
        }
        is CheckoutResult.Error -> {
            println("Checkout failed: ${checkoutResult.message}")
        }
    }
}
```

## Spring Boot Integration

For Spring Boot applications, you can create a configuration class:

```kotlin
import com.plugnpay.sdk.PlugnPayClient
import org.springframework.boot.context.properties.ConfigurationProperties
import org.springframework.boot.context.properties.ConstructorBinding
import org.springframework.context.annotation.Bean
import org.springframework.context.annotation.Configuration

@Configuration
class PlugnPayConfiguration {
    
    @Bean
    fun plugnPayClient(properties: PlugnPayProperties): PlugnPayClient {
        return PlugnPayClient(
            Config(
                apiKey = properties.apiKey,
                apiUrl = properties.apiUrl
            )
        )
    }
}

@ConstructorBinding
@ConfigurationProperties(prefix = "plugnpay")
data class PlugnPayProperties(
    val apiKey: String,
    val apiUrl: String = "https://api.plugnpay.com"
)
```

Then in your `application.yml`:

```yaml
plugnpay:
  api-key: ${PLUGNPAY_API_KEY}
  api-url: https://api.plugnpay.com
```

## Ktor Integration

For Ktor applications, you can integrate PlugnPay in your routing:

```kotlin
import com.plugnpay.sdk.PlugnPayClient
import io.ktor.application.*
import io.ktor.response.*
import io.ktor.routing.*

fun Application.configureRouting() {
    val plugnPayClient = PlugnPayClient.fromEnv()
    
    routing {
        post("/checkout") {
            try {
                val customer = plugnPayClient.getCustomer("customer@example.com")
                val session = plugnPayClient.createCheckoutSession(
                    CreateCheckoutSessionRequest(
                        customerId = customer.id,
                        productId = "premium-plan",
                        successUrl = "https://yourapp.com/success",
                        cancelUrl = "https://yourapp.com/cancel"
                    )
                )
                call.respond(mapOf("checkout_url" to session.url))
            } catch (e: PlugnPayException) {
                call.respond(HttpStatusCode.BadRequest, mapOf("error" to e.message))
            }
        }
    }
}
```
